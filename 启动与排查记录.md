## 项目目录

`/root/.openclaw/workspace/ppt-agent-main`

## 启动方式

### 1. 启动后端

在项目根目录执行：

```bash
cd /root/.openclaw/workspace/ppt-agent-main
PYTHONPATH=backend .venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000
```

后端健康检查：

```bash
curl http://127.0.0.1:8000/healthz
```

预期返回：

```json
{"status":"ok"}
```

### 2. 启动前端

在项目根目录执行：

```bash
cd /root/.openclaw/workspace/ppt-agent-main
npm run dev
```

前端默认监听：

```text
http://localhost:3000/
```

## 排查结论

### 结论 1：项目可以正常启动

本次从该目录启动前后端后，未发现阻止启动的报错。前端和后端都可以拉起，后端接口可正常返回数据。

### 结论 2：`logs/*.pid` 为残留无效 PID

项目中已有：

- `logs/backend.pid`
- `logs/frontend.pid`

但对应 PID 进程实际不存在，且最初检查时 `3000`、`8000` 端口均未监听，因此这些 PID 文件属于历史残留，不能作为服务在线依据。

建议后续排查时不要只看 PID 文件，应配合以下命令确认：

```bash
ss -ltnp | rg ':3000|:8000'
ps -fp "$(cat logs/backend.pid 2>/dev/null)"
ps -fp "$(cat logs/frontend.pid 2>/dev/null)"
```

### 结论 3：存在一个非阻断告警

在访问后端接口时，控制台出现：

```text
FastAPIDeprecationWarning: ORJSONResponse is deprecated
```

这不是启动失败原因，也不影响当前服务运行，但属于后续可以清理的兼容性告警。告警来源位置：

- `backend/app/main.py`

当前代码里 `FastAPI(..., default_response_class=ORJSONResponse)` 仍可运行，但 FastAPI 已不推荐继续这样设置。

## 建议的启动检查流程

每次启动建议按这个顺序检查：

1. 在项目根目录启动后端。
2. 访问 `http://127.0.0.1:8000/healthz` 确认后端健康。
3. 启动前端。
4. 访问 `http://localhost:3000/` 打开页面。
5. 如页面请求异常，再检查 `/api/v1/projects` 是否正常返回。
