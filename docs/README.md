# AI PPT 新环境文档索引

## 1. 文档使用原则

这套文档只描述新环境的目标设计，不描述历史实现，不保留过渡叙事。

文档组织遵守四条规则：

1. 一个业务只归一到一份文档。
2. 提示词和表结构跟着业务走，不再单独散落。
3. 共享概念只在唯一归属文档中定义，其他文档只引用不重复解释。
4. SVG 统一使用 `1280x720`。

## 4. 开发顺序

建议按下面顺序实现：

1. `01-project-workspace.md`
2. `07-agent-orchestration.md`
3. `03-knowledge-research.md`
4. `02-project-init.md`
5. `04-outline.md`
6. `05-page-draft.md`
7. `06-svg-design.md`
8. `08-backend-technical-roadmap.md`
9. `09-mcp-test-deployment.md`

原因很简单：业务边界不先定，技术路线就是空谈；技术路线不最后收口，编码就会发散；运行部署不单独写清楚，联调时一定会反复踩坑。
